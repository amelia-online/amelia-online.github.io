JPL Compiler
As part of my Compilers class, I developed a full compiler for a language called JPL (John Pavel Lang). The compiler included a full lexer, parser, typechecker, and code generation to C and x86 Assembly (NASM) with -O1 and -O3 optimizations. Here is a snippet of the language: 


read image "cat.jpg" to catImg                         
fn modify(img[H, W] : rgba[,]) : rgba[,] {                        
    return array [i : H, j : W] rgba { \
        img[i, j].r * 0.5, \
        img[i, j].g * 0.6, \
        img[i, j].b * 0.3, \
        1.0                \
    }
}
let final = modify(catImg)
write image catImg to "result.jpg"


The code above reads an image and stores it to a variable. It creates a new image by calling the modify function which copies the image but modifies the r, g, and b values. The resulting image is then written to "result.jpg."

HTTP Proxy Server
As part of my Computer Networks class, our first big project was to create an HTTP 1.0 proxy server in Python. The server only supported GET requests, however, the server was able to process multiple clients at once using threads. Additionally, the server cached the responses, and allowed the client to disable the cache or clear the cache. The client could also block websites with a built-in blocklist.
Cpporth

I am not the creator of Porth, and there isn't a lot of documentation on it. I tried to get my interpreter to match the behavior of the actual language as much as possible. Testing using googletest is what I'm working on now.
